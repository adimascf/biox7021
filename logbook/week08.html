Week 08: Trimming assessment on Variant calling and genome assembly

Overview

  • In general, adapter trimming does not appear to offer much benefit at high depth, but the story seems a bit different at lower depth.
  • I think aggregating the metrics on mean or median may be blurring the signal, particularly for the 20x datasets.
  • It might be better to use a delta or log2FC with paired observations, where we compare each trimmed sample directly against its untrimmed. For example, ATCC_25922 20x (untrimmed) versus ATCC_25922 20x (fastplong). This would let us measure the improvement from trimming on a per-sample basis and also control the sample diversity.

Further assessment on variant calling

Looking into false positive and false negative numbers

Extract information about FP and FN, then I sorted them based on sample, then depth, then variant type, then trimmer. Thus, we can compare the stats across the trimmer strightaway per variant type per depth and per sample. I generated interactive table using very cool itables library. We can play around further by sorting and/or filtering the column fields.

Overal, I think FN is the dominant error. In general, our chosen variant calling (Clair3) is very conservative? or it doesnt find the nucleotide that differ?. FP is connsistently low, even at 20x it mostly 1.

If we just look at the absolute number. Some trimmers might be able to save sensitivity at low depth for certain samples (?).

By eyes, for example, in KPC2__202310 20x, fastplong only misses 73 SNP FN, while untrimmed 183. ATCC_10708 20x, porechop_abi_discard does best (88 FN). ATCC_25922 has better metrics if it goes through the trimming step.

But other than that, do not see this pattern in general for all samples.

If we just calculate the median across samples, we loose the paired comparison between each trimmer and the untrimmed control. I think we should compare the numbers within each sample directly, for example KPC2 20x trimmed versus KPC2 20x untrimmed. That way, we keep the paired structure and control better for the diversity across samples

Inspired by plot from Bush et al 2022 , I made plot to show how much “improvement” the trimming tool made per downsampling sample by calculating: Delta_FN = TRUTH_FN (untrimmed) - TRUTH_FN (trimmer):

The calculation is done strictly per sample, per depth, and per variant type.

  • The red dashline is the base line: doing absolutely nothing (untrimmed)
  • Below red dashline: worst than untrimmed
  • Above red dashline: better than untrimmed. It reduced the false negative, can recover variants that would have missed.

My obseration:

  • 100x and 50x points are close to zero, adapter trimming has no impact
  • For SNPs, barbell_default, barbell_max, porechop_abi_discard have a lot of points above the line (rescuing 50-60 SNPs!) at low depth
  • But I think it is still highly unpredictable, because it also can make the FN number worst.
  • Indels more complicated, because a alot of points below red lines for porechop_abi for all configs.

df_plot = data_fnfp.copy()

            ax.legend(title="Depth", loc="upper left", bbox_to_anchor=(1.02, 1))

Code 
import pandas as pd
import matplotlib.pyplot as plt
import seaborn as sns
from pathlib import Path

input_files = list(Path('.').glob("../results/assess/assembly/*/*x/*/*.report.tsv"))

assembly_metrics = []
# extact information that i need which are mismatch per 100kb, indels per 100kbp, and NGA
for file_path in input_files:
    try:
        # extract metadata from the path
        parts = file_path.parts
        trimmer = parts[-4]
        depth = parts[-3]
        sample = parts[-2]
        # index_col=0 sets the metric names as the row index
        df_quast = pd.read_csv(file_path, sep='\t', index_col=0)

        # column name is assembly name
        col_name = df_quast.columns[0]

        # extract the metrics
        mismatches = float(df_quast.loc['# mismatches per 100 kbp', col_name])
        indels = float(df_quast.loc['# indels per 100 kbp', col_name])
        nga = int(df_quast.loc['NGA50', col_name])
        missassemblies = int(df_quast.loc['# misassemblies', col_name])

        assembly_metrics.append({
            'trimmer': trimmer,
            'depth': depth,
            'sample': sample,
            'Mismatches per 100kbp': mismatches,
            'Indels per 100kbp': indels,
            'NGA50': nga,
            'missassemblies': missassemblies
        })

    except Exception as e:
        print(f"Skipping {file_path} due to error: {e}")

df_assembly_metrics = pd.DataFrame(assembly_metrics)


sns.set_theme(style="whitegrid")

metrics = ["Mismatches per 100kbp", "Indels per 100kbp"]
x_col = "trimmer"
hue_col = "depth"

# ordering
order = sorted(df_assembly_metrics[x_col].unique())
hue_order = sorted(df_assembly_metrics[hue_col].unique(),
                   key=lambda x: int(x.replace('x', '')) if 'x' in x else x,
                   reverse=True)

fig, axes = plt.subplots(
    nrows=len(metrics),
    ncols=1,
    figsize=(10.0, 8.0),
    dpi=110,
    sharex=True,
    sharey=False
)

for i, metric in enumerate(metrics):
    ax = axes[i]

    # draw the boxplot (hiding default outliers to avoid duplicates with stripplot)
    sns.boxplot(
        data=df_assembly_metrics,
        x=x_col,
        y=metric,
        hue=hue_col,
        order=order,
        hue_order=hue_order,
        ax=ax,
        fliersize=0,
        gap=0.1
    )

    # overlay the individual points
    sns.stripplot(
        data=df_assembly_metrics,
        x=x_col,
        y=metric,
        hue=hue_col,
        order=order,
        hue_order=hue_order,
        ax=ax,
        alpha=0.6,
        dodge=True,
        linewidth=0.5,
        edgecolor="black",
        legend=False
    )

    ax.set_ylabel(metric)
    ax.set_xlabel("")
    ax.set_title(f"{metric}")
    ax.tick_params(axis="x", labelrotation=45, labelsize=11)

    # manage the legend
    if i == 0:
        ax.legend(title="Depth", loc="upper left", bbox_to_anchor=(1.02, 1))
    else:
        if ax.get_legend() is not None:
            ax.get_legend().remove()

fig.tight_layout()
plt.show()

NGA50

Because NGA50 values are huge, I normalise to percentage value. NGA50 Change (%) = ((NGA50 (trimmer) - NGA50 (untrimmed)) / NGA50 (untrimmed))*100%

  • Above the red line: Trimmer’s NGA50 is longer than untrimmed
  • Below the red line: Trimmer’s NGA50 is shorter than untrimmed

My observation:

  • Adapter trimming does not improve contiguity or structural accuracy at al at 100x and 50x depth. In fact, almost all the deviation at 50x and 100x go down!
  • at low depth 20x, some trimmer can improve certain samples, other can make it worst. For example porechop_abi_discard improve NGA for particular sample ATCC_25922__202309 20x by ~900%! (look at the raw table below)
  • barbell also improve this sample ATCC_10708__202309 at the low depth by more than 100%!
  • barbell_default worsen BPH2947__202310 at 100x depth! from 3049603 to 1709888. Overall, this trimmer damage the dataset.
  • barbell_max worsen MMC234__202311 at 100x depth!. Here, I just relised the visualisation hides the actual damage of the trimming process for NGA50 metrics 🥲
