Week 13: Visualisation exploration and Quality based tools assessment

Overview

I made pointplot figures for the F1 metrics. I ran into a weird issue while creating them, but managed to find a workaround.

I also explored different visualisation approaches, including pointplot, stripplot + boxplot, and barplot, as well as using either the median or mean as the point/bar height for each assessment metric we generated.

I still do not see any clear winner or loser among the tools, or any tool that consistently performed better than the untrimmed dataset.

Selected and ran a representative seq of quality based preprocessing tools with their respective paramter sets. The, I compiled and plotted the resulting results.

Weekly meeting and action items (27-05-2026)

  • Issue with logit scalling on the y axis and average calculation in the poinplot/barplot. Also
  • My current workaround
  • Median vs mean? I personally prefer using the median because most of our data is highly skewed (towards 1 or towards 0). Also, median is resistant to outliers. However, in some cases we can observe “pattern” more clearly using the mean compared to no pattern on median.
  • Normalised NGA50 value
  • For sanity check, based on rasusa downsampling log all samples after quality filtering were above 99.96x (~100x) depth.
  • Filtlong default (which prefers longer and higher-qscore reads) is the best

After spending so much time debugging how and why, from reviewing my snakmake code until looking on my matplotlib + seaborn code, it seems y axes sharing + log scale on y axes like on the plot above make the average calculation in the poinplot (also in barplot) a bit funky. So my workaround, I just turn off the sharey=False. This makes the y axes is independent for each facet, and make the points now match the value in the table. This issue doesn’t seem appear when we use median in the poinplot.

F1 score using pointplot (median)

Code 
import sys
import pandas as pd
import matplotlib.pyplot as plt
import seaborn as sns
import numpy as np

input_csv = "../results/tables/assess/call-alltrim/metrics/trimming_variant_summary.csv"

sns.set_theme(style="whitegrid")

pr_df = pd.read_csv(input_csv)

metrics = ["F1_SCORE"]
vartypes = ["SNP", "INDEL"]
models = ["sup", "hac"]

dataix = pr_df.groupby(["trimmer", "depth", "VAR_TYPE", "sample", "model"])["F1_SCORE"].idxmax()
data = pr_df.iloc[dataix].copy()

order = sorted(data["trimmer"].unique())
hue_order = ["100x", "50x", "20x"]

for y in metrics:
    fig, axes = plt.subplots(
        nrows=len(vartypes),
        ncols=len(models),
        figsize=(14, 10),
        dpi=300,
        sharex=True,
        sharey=True,
    )

    for r, vartype in enumerate(vartypes):
        for c, model in enumerate(models):
            ax = axes[r, c]
            df = data.query("VAR_TYPE == @vartype and model == @model").copy()

            cap = 0.99999
            df.loc[:, y] = df[y].apply(lambda v: cap if v > cap else v)
            yticks = [0.01, 0.1, 0.5, 0.8, 0.9, 0.99, 0.999, 0.9999, cap]
            yticklabels = [f"{yval:.2%}" for yval in yticks]

            if not df.empty:
                sns.pointplot(
                    data=df, x="trimmer", y=y, hue="depth",
                    order=order, hue_order=hue_order,
                    ax=ax,
                    dodge=0.3, # separates the hues so they don't overlap
                    errorbar=('ci', 95), # default
                    capsize=0.1, # adds caps to the error bars
                    err_kws={'linewidth': 1.5},
                    estimator='median'
                )

            ax.set_yscale("logit", nonpositive="clip")
            ax.set_yticks(yticks)
            ax.set_yticklabels(yticklabels)

            if c == 0:
                ax.set_ylabel(f"{vartype} F1 score")
            else:
                ax.set_ylabel("")

            if r == 0:
                ax.set_title(f"{model} model", fontsize=14)
            else:
                ax.set_title("")

            ax.set_xlabel("")
            if r == len(vartypes) - 1:
                ax.tick_params(axis="x", labelrotation=45, labelsize=10)
            else:
                ax.tick_params(axis="x", bottom=False)

            ax.set_xticks(range(len(order)))
            ax.set_xticklabels(order, rotation=45, ha="right", rotation_mode="anchor", fontsize=11)

            # Legend Handling: Show it only once, on the top-right plot
            if r == 0 and c == 1:
                if ax.get_legend() is not None:
                    ax.legend(title="Depth", bbox_to_anchor=(1.05, 1), loc='upper left')
            else:
                if ax.get_legend() is not None:
                    ax.get_legend().remove()

    fig.tight_layout()
    fig.savefig('../results/figures/assess/call-alltrim/metrics/trimming_variant_f1_pointplot_median.png')

Quality based processing tools

Tools Parameters What does the command do?
Fastplong_1 fastplong -A -l 100 -m 15 Disables adapter trimming; keeping only reads >= 1000bp and mean Q >= 15
Fastplong_2 fastplong -A -l 1000 -m 15 Disables adapter trimming; keeping only reads >= 1000bp and mean Q >= 15
Filtlong_1 filtlong --length_weight 0 --window_q_weight 0 --target_bases [100x] Targets specific coverage based only on quality score (ignoring length)
Filtlong_2 filtlong --mean_q_weight 0 --window_q_weight 0 --target_bases [100x] Targets specific coverage based only on length (ignoring quality)
Filtlong_3 filtlong --target_bases [100x] Targets specific coverage using default weighting (balances both read length and quality)
Chopper_1 chopper --trim-approach trim-by-quality 15 -l 1000 Trims low quality bases from the read ends until reaching a base Q >= 15, removing reads < 1000bp
Chopper_2 chopper --trim-approach trim-by-quality --cutoff 15 -l 100 Trims low quality bases from the read ends until reaching a base Q >= 15, keeping reads >= 100bp
Chopper_3 chopper --trim-approach best-read-segment --cutoff 15 -l 1000 Extracts the highest quality continuous segment Q >= 15 from the read, keeping reads >= 1000bp
Chopper_4 chopper --trim-approach best-read-segment --cutoff 15 -l 100 Extracts the highest quality continuous segment Q >= 15 from the read, keeping reads >= 100bp
Seqkit_1 seqkit seq -m 100 -Q 15 Keeping re–cutoff ads >= 1000bp and mean Q >= 15
Seqkit_2 seqkit seq -m 1000 -Q 15 Keeping re–cutoff ads >= 1000bp and mean Q >= 15
  • Q set to 15 for SUP, 10 for HAC
  • [100x] = 100x of the genome size
