Contamination Assessment

Contamination logic in Barbell paper

Based on their script in barbell eval, using python binding sassy-rs, they scan rapid barcode and flank sequences in their polished genome assembly results.

The sequence structure they searched for as the “contaminants” is like this: GCTTGGGTGTTTAACC - 24bp BARCODE SEQUENCE - GTTTTCGCATTTATCGTGAAACGCTTTCGCGTTTTTCGTGCGCCGCTTCA

Their fasta is here in their repository, and the information of this left flank - barcode right flank sequences is here on ONT doc.

The set k = 20 in their script, matches up to (and including) 20bp distance threshold. Macthes are then reported as contaminants.

Adapted from their script, I wrote this script to detect barcode contaminants in our tested assembly result. Before i tested in our assembly, I tested the script on their assembly result where they found three contaminants on it. I manually demultiplexed their fastq file, and ran their assembly pipeine to get theit BC49 assembly.

It does match their finding where they found 3 contaminants hits in the BC49, including the 90bp contamination with 0 distance in the first 90 in the contig.

I also did notice, they mention that they were using SQK-RBK110.96 kit in the paper, but they specify SQK-RBK114.96 kit for dorado demux command in their script. Not sure how much this discrepancy impact demultiplexing overall, but in my case, it does make difference on how many hits found.

With the “correct” kit paramter SQK-RBK110-96 for dorado demux, using the same script, I only found one hit (the hit with 0 distance)

#!python detect_adapter_contamination_cli.py -a bc49_medaka_consensus.110.fasta -b rapid_flanks_w_bars.fasta -o bc49_contam.110.tsv
!csvtk pretty -H bc49_contam.110.tsv
bc49_medaka_consensus.110.fasta BC49 contig_19 1 91 0 rapid_flanks_w_bars.fasta

If I understand correctly, they used k = 20 here to detect barcode contaminations in the assemblies as they also do when they detect barcode they want to trim it and demux the reads. Using this formula:

\[\theta_{\text{emp}}(a) = \max\left(0,\; 0.51a - 1.7312\sqrt{a}\right)\]

where (a) is the effective length, calculated as:

\[a = L_{\text{barcode+flank}} - L_{\text{barcode}}\]

Here, total barcode-plus-flank length s constant across samples, while the barcode length vary between samples.

In the our contamination detection script, I used the formula to automatically set k (distance) whther we scan for native or rapid fasta sequences also from barbell repo here.

Note: Before i ran the script in our assembly results, I am a bit concerned that their and also our contamination assessment may not fully independent (basically the same logic) from Barbell’s demultiplexing (to detect and trim trim, then assign the reads) step. Using this assessment, I think barbell processed data would be guaranteed to show no barcode contamination.

import pandas as pd
import matplotlib.pyplot as plt
import seaborn as sns

contamination_count = pd.read_csv("../results/tables/assess/assembly/metrics/contaminant_summary_count.csv")

sns.set_theme(style="whitegrid")
models = ["sup", "hac"]
hue_order = ["100x", "50x", "20x"]
order_abs = sorted(contamination_count["trimmer"].unique())

fig, axes = plt.subplots(nrows=1, ncols=len(models), figsize=(14, 5.5), dpi=110, sharex=True, sharey=True)

for c, model in enumerate(models):
    ax = axes[c]
    # During loop 1, the variable `model` equals "sup"
    contamination_count_sub = contamination_count.query("model == @model")

    if not contamination_count_sub.empty:
        sns.boxplot(data=contamination_count_sub, x="trimmer", y="contamination_count", hue="depth", order=order_abs, hue_order=hue_order, ax=ax, fliersize=0, gap=0.1)

        sns.stripplot(data=contamination_count_sub, x="trimmer", y="contamination_count", hue="depth", order=order_abs, hue_order=hue_order, ax=ax, alpha=0.6, dodge=True, linewidth=0.5, edgecolor="black", legend=False)

    # force standard numbers instead of scientific notation
    ax.yaxis.set_major_formatter(plt.ScalarFormatter())

    # clean up the labels for a polished look
    ax.set_ylabel("Contamination Count" if c == 0 else "")
    ax.set_title(f"Number of Contaminants - {model.upper()} Model")
    ax.set_xlabel("")
    ax.tick_params(axis="x", labelrotation=45, labelsize=11)

    if c == 1 and ax.get_legend() is not None:
        ax.legend(title="Depth", bbox_to_anchor=(1.05, 1), loc='upper left')
    elif ax.get_legend() is not None:
        ax.get_legend().remove()

fig.tight_layout()
plt.show()

Mean and median of contamination count

Code 
import pandas as pd
from itables import show


# broup by trimmer, model, and depth, then calculate multiple statistics across the samples
summary_stats = contamination_count.groupby(["trimmer", "model", "depth"])["contamination_count"].agg(
    Mean='mean',
    Median='median',
    Max='max'
).reset_index()

# Round them
summary_stats['Mean'] = summary_stats['Mean'].round(2)

# Apply ythe sorting logic so the table is perfectly organised
depth_order = pd.CategoricalDtype(categories=["100x", "50x", "20x"], ordered=True)
model_order = pd.CategoricalDtype(categories=["sup", "hac"], ordered=True)

summary_stats['depth'] = summary_stats['depth'].astype(depth_order)
summary_stats['model'] = summary_stats['model'].astype(model_order)

# Sort: Trimmer -> Model -> Depth
summary_stats = summary_stats.sort_values(by=["trimmer", "model", "depth"])

summary_stats.reset_index(drop=True, inplace=True)
show(summary_stats, classes="display", scrollX=True, paging=True, lengthMenu=[10, 20, 50])
